Development

The Agent-Led Workflow.

Built entirely through CLI interactions. The agent was the architect, the developer, and the debugger.

The Blueprint

We used Claude Haiku to draft the plan, then used Gemini CLI to build the repository and the app's foundation.

AI Instruction Manuals

We used specialized files (OKKIO.md & GROOVSTER.md) to teach the AI our team's roles and rules for each feature.

Our Team Strategy

  • Vertical Slices: Groove owned Notes, Okkio owned Tasks
  • Instruction-Driven: The AI followed our team protocols exactly
  • GitHub CLI: Managed our workflow and branches seamlessly

The Collaboration

Expert Hand-offs.

We didn't just use one AI; we picked the best expert for every specific job.

The Power of Choice

We used Gemini for setup, Haiku for strategy, and Codex for solving the deepest coding problems.

Human-Guided Building

We used terminal feedback to guide the AI, letting it "see" its mistakes so it could fix them and keep building.

Strategic Engineering

  • Team Effort: Strategic hand-offs between specialists
  • Automated Checks: We used "Memories" to make the AI run the linter after every change
  • Manual Precision: Human oversight at every critical step

Optimization

Smart Context Management.

We kept the AI fast and efficient by manually controlling the information it held in its "mind."

Fresh Sessions

We started new chat sessions for each task to keep the context clean and prevent the AI from getting confused.

Caveman Mode

With Codex, we used "Caveman Mode"—ultra-short, compressed language—to save tokens and get more work done for free.

Our Efficiency Tricks

  • Starting fresh chat windows to stay sharp and fast
  • Using "Caveman" talk to maximize our free token budget
  • Only reading the exact lines of code that mattered
